Interactive Animation for Methodist Centre

The Methodist Centre (MC) provides social services to many groups, including children, youth, adults, families, elderly people, ethnic minorities, schools, and companies. Their services include child and family support, youth programs, elderly health care, community help, counseling, special education needs (SEN) support, school partnerships, continuing education, and job training. MC works closely with communities to adapt their services and meet changing needs over time.

CHALLENGES AND SOLUTIONS

This is more than just a product design case study. I want to share how I used Rive to create an interactive animation for the Methodist Centre (MC) project. For the MC website, another team member was primarily responsible for the UI, while I handled animations and some minor UI elements. The client requested an animation for the landing page, and the team lead showed me a reference animation, asking, "Can you create something similar for the client?"

Image: Initial vector illustration before animation creation.

At first, I was tasked with creating the animation in SVGator. But after reviewing the reference, I realized it required interactive triggers, not just animation. As you may know, SVGator allows you to create animations within its editor, but developers need to use JavaScript event listeners during the coding stage to add interactions. This meant extra work because the animation had to be created first, then the triggers had to be implemented separately.

I started looking for a tool that could handle both SVG animation and interaction triggers efficiently in one place. That was when I thought of Rive, a tool that uses State Machine logic. I explained to the team that using SVGator would make the process more complicated and presented Rive as a better alternative. After discussing the pros and cons, we decided to use Rive.

Image: Supported Platforms for Rive Runtimes

Let me explain a bit about Rive. It has a timeline-based animation system and supports both vector and raster graphics. For vectors, you can animate basic properties like position, scale, and rotation, and even edit vertices for morphing animations. For raster graphics, meshes are used for morphing effects.

Animation timelines in Rive are called States, and you can create multiple ones, such as Idle, Jumping, or Running for a character. These states are placed into the State Machine and connected with transitions. To control state transitions, you use inputs like numbers, booleans, and triggers. For example, setting isRunning to true allows you to transition from Idle to Running. You can also set conditions, such as "If isJumping is true, transition from Running to Jumping." Additionally, Rive has an Event feature that can send signals outside the animation. All these elements, including States, Transitions, Inputs, Conditions, and Events, are managed within the State Machine.

Image: Animated timeline for Buildings

For the MC animation, we used all these concepts. The main requirement was to represent MC's four main services, which are Health Care, Youth, Family and Community Care, Talent Care, and Elder Care, as four buildings. The buildings needed to highlight on mouse hover and direct users to external links when clicked. The animation also included additional elements to enhance the experience.

Image: State Machine setup for MC Animation

One of the biggest challenges was handling the highlighted layer of service buildings during mouse hover. When highlighted, the layer would move to the top, causing vehicles in the animation to appear as if they were driving underneath the buildings. This looked unnatural, so we had to take additional steps to fix it. After troubleshooting different layering techniques, we finally found a solution that maintained the intended animation effect without visual inconsistencies.

Another interesting aspect of this project was implementing external links when clicking on service buildings. At the time, Rive did not have an events function, so I was unsure how to achieve this. Fortunately, just a few days after starting the project, Rive released a new version that introduced the events function with three types: General, Open URL, and Audio. This update allowed us to add external link triggers directly within the animation, making implementation much smoother.

Image: Web (JS) Runtime

Unlike SVG animations, Rive animations cannot run directly when embedded. Instead, Rive provides an open-source library called Rive Runtime, which can be integrated with various platforms. The website's hero section animation was also created using Rive.

You can view and interact with the MC animation, which is the main subject of this case study, at this link: https://vv7n94.csb.app/.
